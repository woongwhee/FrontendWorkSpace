자바스크립트 개용.

1.스크립트란?

프로그램의 동작을 사용자(클라이언트)의 요구에 맞게 수행되도록 해주는 용도
매우 빠르게 배우고 짧은소스코드로 사용하도록 고안된것이다!

2. 자바스크립트란?

웹브라우저에서 가장 많이, 가장 기본적으로 사용되는 "인터프리터 방식"의 스크립트 언어다. - 자바: 컴파일 방식 => 컴파일을 거치고 나서 실행이 된다.
- 자바스크립트 : 인터프리터 방식 => 컴파일을 거치지 않고 곧바로 실행된다. (브라우저에 html문서가 로딩 될때 코드를 한줄씩 읽어내는 방식.)


3. 장단점

* 장점

  • 요소에 이벤트가 발생했을때 코드를 한줄씩 읽어나가면서 수행되기 때문에 수행속도가 빠름
  • 간단한 코드 작성으로 초보자가 접하기 쉬운언어
  • 접근제한자 개념이 없음. 메솓 정의시 반환형 정의안함 변수선언이 쉬움.

* 단점

  • 웹에 특화된 기술이라 내부에서 제공되는 기능이 제한적임
  • HTML 내에 소스코드를 작성하게되면 외부에 공개됨 => 보안에 취약점.

4. 자바스크랩트 소스코드 작성위치에 따른종류

  1. inline: 이벤트를 부여하고자 하는 요소내에 작성하는 방식
  2. internal : 해당html문서내에 script영역에 소스코드 작성하는 방식
  3. external:소스코드들을 별도의 .js 파일로 작성해서 가져다가 실행,

1)inline

태그내에 직접적으로 실행할 간단한 소스코드에 작성해서 실행시킴 주로 실행해야 하는 소스가 매우 소량일 경우 사용

2)internal

script 태그 영역에 함수단위로 소스코드들을 작성해놓고
어떤요소에 어떤 이번트가 발생시 해당 그 함수를 호출함으로써 실행시키는 방식
- script태그는 head body에 모두작성가능

3)external

별도의 .js파일로 소스코드를 작성하고 해당 html에 가져다가 실행하는 방식.
